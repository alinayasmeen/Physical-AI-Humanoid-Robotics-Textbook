# لیب 1: گیزبو میں فزکس تجربات

**موڈیول**: 2 - ڈیجیٹل ٹوئن
**مدت**: 60 منٹ
**پریشانی**: مبتدی

## اہداف

گیزبو سیمولیشن میں مختلف فزکس پیرامیٹر سیٹنگز کے تحت روبوٹ کے رویے کا مشاہدہ اور تجزیہ کریں. گریویٹی، فرکشن، اور ماس کی خصوصیات کو تبدیل کرکے، آپ سیکھیں گے کہ فزکس انجن روبوٹ موشن کو کیسے متاثر کرتے ہیں.

## ضروریات

- موڈیول 1 (ROS 2 بنیادیات) مکمل
- موڈیول 2 لیسن 1 (ڈیجیٹل ٹوئن تصورات) مکمل
- ROS 2 ہمبل کے ساتھ اوبنٹو 22.04 انسٹال
- گیزبو فورٹریس انسٹال (`ros-humble-ros-gz`)

## سیکھنے کے نتائج

اس لیب کو مکمل کرنے کے بعد، آپ:
1. ایک سادہ روبوٹ ماڈل کے ساتھ گیزبو لانچ کریں گے
2. ڈیفالٹ سیٹنگز کے تحت فزکس کا رویہ دیکھیں گے
3. گریویٹی میں تبدیلی کریں گے اور تبدیلیاں دیکھیں گے
4. فرکشن کوائف میں تبدیلی کریں گے اور تبدیلیاں دیکھیں گے
5. بعد کے حوالہ کے لیے مشاہدات دستاویز کریں گے

## سیٹ اپ

### 1. انسٹالیشن کی تصدیق کریں

```bash
# چیک کریں کہ گیزبو انسٹال ہے
gz sim --version

# چیک کریں ros_gz برج
ros2 pkg list | grep ros_gz
```

### 2. لیب ڈائریکٹری میں جائیں

```bash
cd ~/ros2_ws/src  # یا آپ کا ورک سپیس
cp -r /path/to/labs/module-02/lab-01-physics-experiments .
cd lab-01-physics-experiments
```

## لیب اسٹیپس

### اسٹیپ 1: ڈیفالٹ سیمولیشن لانچ کریں (10 منٹ)

ایک سادہ گرتی ہوئی کیوب کے ساتھ گیزبو لانچ کریں:

```bash
# ٹرمنل 1: ڈیفالٹ فزکس کے ساتھ گیزبو لانچ کریں
gz sim -r starter/falling_cube.sdf
```

**مشاہدہ کریں**:
- کیوب کتنا تیزی سے گرتا ہے؟
- زمین سے ٹکرانے پر کیا ہوتا ہے؟
- جب آپ اسے دھکیلیں (فرس لگانے کے لیے گیزبو GUI استعمال کریں)؟

**ذیل کی میز میں اپنے مشاہدات ریکارڈ کریں**.

### اسٹیپ 2: گریویٹی میں تبدیلی کریں (15 منٹ)

ورلڈ فائل میں گریویٹی سیٹنگز تبدیل کرنے کے لیے ایڈٹ کریں:

```bash
# اسٹارٹر فائل کھولیں
nano starter/falling_cube.sdf
```

`<gravity>` ٹیگ تلاش کریں اور یہ قیمتیں آزمائیں:

| تجربہ | گریویٹی (m/s²) | متوقع رویہ |
|------------|----------------|-------------------|
| زمین | 0 0 -9.8 | نارمل فول سپیڈ |
| چاند | 0 0 -1.62 | سست، فلوٹنگ فول |
| مریخ | 0 0 -3.71 | معتدل فول سپیڈ |
| مشتری | 0 0 -24.79 | تیز، بھاری فول |
| صفر-جی | 0 0 0 | آبجیکٹ فلوٹ کرتا ہے |

ہر سیٹنگ کے لیے:
1. فائل محفوظ کریں
2. گیزبو دوبارہ شروع کریں: `gz sim -r starter/falling_cube.sdf`
3. مشاہدہ کریں اور رویہ ریکارڈ کریں

### اسٹیپ 3: فرکشن میں تبدیلی کریں (15 منٹ)

زمینی سطح کے لیے فرکشن کوائف (`mu`) تبدیل کریں:

```xml
<!-- زمینی ماڈل میں -->
<surface>
  <friction>
    <ode>
      <mu>0.1</mu>   <!-- کم فرکشن (برف) -->
      <mu2>0.1</mu2>
    </ode>
  </friction>
</surface>
```

یہ فرکشن قیمتیں ٹیسٹ کریں:

| سطح کی قسم | mu قیمت | متوقع رویہ |
|--------------|----------|-------------------|
| برف | 0.1 | آبجیکٹ آسانی سے سلائیڈ کرتا ہے |
| لکڑی | 0.4 | معتدل سلائیڈنگ |
| کنکریٹ پر ربر | 1.0 | زیادہ گرپ، کم سلائیڈنگ |
| چپچپا | 2.0 | تقریبا کوئی سلائیڈنگ نہیں |

### اسٹیپ 4: فزکس آبزرور اسکرپٹ چلائیں (10 منٹ)

پروگرامی طور پر فزکس کا مشاہدہ کرنے کے لیے فراہم کردہ پائی تھن اسکرپٹ استعمال کریں:

```bash
# ٹرمنل 1: گیزبو لانچ کریں
gz sim -r starter/falling_cube.sdf

# ٹرمنل 2: آبزرور چلائیں
python3 starter/physics_observer.py
```

اسکرپٹ کرے گا:
- کیوب کے پوز کو سبسکرائب کرے گا
- فول ویلوسٹی کا حساب کرے گا
- فزکس کے مشاہدات رپورٹ کرے گا

### اسٹیپ 5: حل کے ساتھ موازنہ کریں (10 منٹ)

حل فائلز کا جائزہ لیں:

```bash
# اپنے مشاہدات کو متوقع نتائج کے ساتھ موازنہ کریں
cat solution/expected_observations.md

# مکمل حل اسکرپٹ چلائیں
python3 solution/physics_observer.py
```

## ریفلیکشن سوالات

1. **چاند کی گریویٹی (1.62 m/s²) ہیومنوائڈ روبوٹ کے چلنے پر کیا اثر ڈالے گی؟**
   - توازن، قدم کا وقت، توانائی کے استعمال کے بارے میں سوچیں

2. **زیادہ گریویٹی زیادہ جوائنٹ تناو کیوں پیدا کرتی ہے؟**
   - وزن اور ٹورک کے تعلق کو مدنظر رکھیں

3. **فرکشن کوائف ایک چکروں والے روبوٹ بمقابلہ ایک لیگ والے روبوٹ کو کیسے متاثر کر سکتا ہے؟**
   - مختلف لوکوموشن اقسام کے لیے ٹریکشن کی ضروریات کا موازنہ کریں

4. **بہت زیادہ فرکشن قیمتوں کے ساتھ سیمولیشن اسٹیبلٹی پر کیا ہوتا ہے؟**
   - عددی درستی اور فزکس انجن کی حدود کو مدنظر رکھیں

## مسئلہ حل

### گیزبو لانچ نہیں ہوتا
```bash
# چیک کریں کہ کوئی اور انسٹینس چل رہا ہے
pkill -9 gz

# کیش صاف کریں
rm -rf ~/.gz/
```

### برج کنکٹ نہیں ہو رہا
```bash
# چیک کریں کہ ros_gz انسٹال ہے
sudo apt install ros-humble-ros-gz
```

### فزکس غیر متوقع طریقے سے رویہ رکھتا ہے
- چیک کریں کہ SDF ورژن گیزبو ورژن سے مماثل ہے
- یقینی بنائیں کہ تمام ضروری پلگ ان لوڈ ہیں
- یقینی بنائیں کہ فزکس پیرامیٹر مناسب حدود میں ہیں
